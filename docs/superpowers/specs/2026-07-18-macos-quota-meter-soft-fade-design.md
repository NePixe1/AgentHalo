# macOS 配额条柔和淡化设计

## 背景

macOS 详情面板的配额条目前使用固定青蓝色 `#4AAFC9`。用户希望配额充足时采用 Windows 配额条当前使用的深色 `#406984`，并在剩余额度降低时逐渐变淡。视觉方案已选定为「B. 柔和淡化」：中高额度保持较强对比，接近用尽时淡化更明显。

## 范围

- 仅修改 `src/macos` 的配额条填充颜色。
- Windows 端保持现状，不修改 `src/windows`。
- 配额条宽度、4pt 高度、圆角、背景轨道颜色和详情面板整体布局保持不变。
- 继续使用“剩余百分比”同时驱动填充长度与填充颜色。
- 不修改配额解析、月度/Plus 展示选择、刷新时间或文案。

## 颜色映射

`RoundedMeterView.value` 继续表示 `0...100` 的剩余百分比。绘制前先将输入钳制到这个范围，然后按照以下色阶锚点进行分段线性 RGB 插值：

| 剩余额度 | 填充色 |
| ---: | :--- |
| 100% | `#406984` |
| 75% | `#527992` |
| 50% | `#7094A9` |
| 25% | `#A8BFCA` |
| 0% | `#CAD9E0` |

该映射连续变化，不会在锚点处跳色。高额度区间的色差较小，保留墨色的视觉重量；低额度区间的色差增大，使临近用尽的状态明显更淡。0% 时沿用现有行为，不绘制填充，因此 0% 锚点只用于接近零额度时的连续插值。

## 组件与数据流

在 macOS 详情面板代码中提取一个无副作用的配额颜色计算函数：输入剩余百分比，输出对应 `NSColor`。`RoundedMeterView.draw(_:)` 仍负责绘制背景轨道和填充几何，仅把当前固定填充色替换为计算结果。

现有数据流保持不变：

```text
RateLimitSnapshot usedPercent
  → QuotaRowView.update
  → remaining = 100 - usedPercent
  → RoundedMeterView.value
  → 填充长度 + 柔和淡化颜色
```

`No Data`、等待刷新和已过期配额仍将 meter 值设为 0，因此只显示背景轨道，不出现浅色残留条。

## 测试与验收

先在 `HaloInteractionChecks.swift` 中增加会失败的回归检查，再实现颜色映射。检查应覆盖：

1. 100%、75%、50%、25% 分别得到已确认的锚点颜色。
2. 0% 不绘制填充，并验证颜色函数的 0% 锚点。
3. 小于 0 和大于 100 的输入会被安全钳制。
4. 两个锚点之间的颜色连续插值，且剩余额度降低时三个 RGB 通道均不变暗。
5. 配额条现有填充长度、4pt 高度、圆角、背景轨道和详情面板高度不变。
6. Windows 文件无改动。

完成后从 `src/macos` 运行 `swift run AgentHaloMac --self-check`、`swift run AgentHaloCoreChecks` 和 `swift build`，再从仓库根目录运行打包验证，最终以 `outputs/AgentHalo-macOS/AgentHalo.app` 的可见效果为准。
